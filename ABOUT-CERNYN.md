# Sobre a Cernyn

A **Cernyn** é uma consultoria de inteligência artificial. Ajudamos executivos e empresas a transformar ideias em produtos com IA — com estratégia, engenharia e governança.

🌐 **Site oficial:** [cernyn.com](https://cernyn.com/)

---

## O que fazemos

Trabalhamos lado a lado com executivos para:

- **Estratégia de IA** — identificar onde IA cria valor real, sem hype
- **POCs e protótipos** — validar hipóteses com baixo custo e prazo curto
- **Engenharia de produto com IA** — construir produtos que escalam, com qualidade e segurança
- **Governança e responsabilidade** — dados sensíveis, LGPD, observabilidade, custos previsíveis
- **Capacitação de times** — treinar equipes técnicas e de negócio para usar IA com autonomia

Atendemos empresas que querem **resultado prático** com IA, não vitrine de tecnologia.

---

## Por que criamos este scaffold

Este repositório (`padrao-claude-code-cernyn` ou similar) é uma **destilação pública** das nossas convenções de engenharia para experimentos com Claude Code conduzidos por **executivos não-desenvolvedores**.

Observamos um padrão recorrente nas empresas que apoiamos:

1. Um diretor ou superintendente tem uma ideia
2. Começa a prototipar com Claude Code (ótimo!)
3. Escolhe stack/práticas que não casam com o resto da empresa
4. Quando o experimento "dá certo" e precisa virar produto, o time de dev tem que **reescrever do zero**

Este scaffold resolve o problema na origem: o experimento já nasce com **boas práticas e stack alinhado com o mercado corporativo**, de modo que a internalização pelo time de dev seja rápida e sem dor.

## Quem somos para você

- **Você é executivo de uma empresa?** Use este scaffold à vontade. É gratuito e aberto. Se precisar de apoio especializado (estratégia, implementação, capacitação), fale com a gente.
- **Você é dev recebendo um experimento?** Este scaffold foi feito pensando no seu trabalho — para que o que chega até você esteja minimamente alinhado com práticas de engenharia. Issues e PRs são bem-vindos.
- **Você é consultor/empresa parceira?** Se quiser usar este scaffold com seus clientes, fique à vontade — só pedimos que mantenha a atribuição à Cernyn.

## Princípios da Cernyn

Quatro princípios que guiam o que fazemos:

1. **Pragmatismo sobre dogma** — IA não é solução universal. Indicamos quando *não* usar.
2. **Direcionamento, não imposição** — sugerimos o caminho certo; quem decide é você.
3. **Engenharia desde o experimento** — boas práticas valem mesmo em POC.
4. **Negócio em primeiro lugar** — métrica de sucesso é valor para o cliente, não elegância técnica.
